故事地图适合先从“我要讲什么”开始，而不是从“我有哪些字段”开始。你更需要的是主线、章节、关键地点和页面气质，而不是一开始就准备大量表格。

## 推荐节奏

<Steps>

1. 先确定故事主线。

   想清楚你想表达的主题、地点、人物、时间线，以及希望用户读完后记住什么。

2. 先让系统生成第一版结构。

   第一次重点描述页面该怎么组织，例如标题区、故事面板、地图联动、章节切换、图例等。

3. 逐轮补叙事细节。

   在第一版基础上继续补文案语气、重点地点、路线节点和视觉风格，不必一次写满所有要求。

4. 强调地图和内容联动。

   明确要求章节切换、地点卡片和地图视角发生联动，而不是只在页面上放一张底图。

5. 最后再修正文案和层次。

   页面结构稳定后，再继续提出“更克制”“更正式”“更适合汇报”这类修改要求。

6. 达到展示状态后再发布。

   分享之后仍可以继续维护标题、描述和公开状态。

</Steps>

## 故事地图最值得先说明的四件事

<CardGrid>
  <Card title="主题是什么" icon="book">
    先回答“这个页面到底在讲什么”，不要只说“做一个地图”。
  </Card>
  <Card title="章节怎么展开" icon="list">
    如果是时间线、路线或多个地点，最好先说出章节顺序。
  </Card>
  <Card title="关键地点有哪些" icon="pin">
    哪些点需要重点放大、哪些点只需要轻量说明，最好先分层。
  </Card>
  <Card title="页面气质是什么" icon="paintbrush">
    例如庄重、克制、叙事感强、适合汇报、适合传播等。
  </Card>
</CardGrid>

## 可直接复用的故事地图提示词

```text
请做一个故事地图页面，主题是“北京中轴线的历史与空间演变”。
页面需要有：
1. 一个清晰的标题区；
2. 左侧章节导航和地点说明；
3. 右侧地图跟随章节切换联动视角；
4. 风格克制、正式，适合汇报展示；
5. 重点地点要有简短说明卡片。
```

如果第一版已经有了雏形，不要轻易推翻。通常继续补“章节节奏、重点地点说明、地图镜头联动、页面气质”就能越改越稳。
