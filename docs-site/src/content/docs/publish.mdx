当页面接近交付状态后，发布并不是最后一个动作。真正完整的分享流程，还包括标题、描述、公开方式和后续维护。

## 发布前先想清楚两件事

1. 这个页面是给内部流转、审稿和预览用，还是给外部展示和传播用。
2. 发布后谁负责继续维护标题、描述和公开状态。

## 可见性选择

<Tabs>
  <TabItem label="未公开链接">
    只有拿到链接的人可访问，适合内部试阅、审稿和项目协作。第一次发布通常更推荐先用这个方式。
  </TabItem>
  <TabItem label="公开样例">
    页面会进入公开样例集合，适合做能力展示、模板积累或对外案例呈现。
  </TabItem>
</Tabs>

## 标准发布步骤

<Steps>

1. 确认页面已经达到可展示状态。

   至少确保地图能正常加载、主要内容完整、文案没有明显错误。

2. 填写标题和描述。

   标题建议能明确主题，描述建议说明页面用途、范围或观看方式。

3. 选择公开方式。

   内部预览建议先用未公开链接，对外展示再切为公开样例。

4. 发布后继续维护。

   如果用途发生变化，创建者仍然可以继续修改分享信息，不必重新生成页面本体。

</Steps>

## 三种典型状态

<CardGrid>
  <Card title="未公开链接" icon="lock">
    最适合项目内部沟通、试阅、审稿和迭代过程中的确认。
  </Card>
  <Card title="公开样例" icon="open-book">
    更适合对外展示、案例积累和宣传传播。
  </Card>
  <Card title="创建者继续维护" icon="pencil">
    分享创建者仍可修改标题、描述和可见性，让页面始终与最新用途保持一致。
  </Card>
</CardGrid>

<Aside type="tip" title="交付现场最稳妥的做法">
正式对外展示前，先把分享链接发给一位内部同事试阅，确认标题、描述、首屏内容和访问方式都符合预期，再切换为公开状态。
</Aside>
