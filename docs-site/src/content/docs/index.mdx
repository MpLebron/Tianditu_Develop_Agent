这套文档面向第一次接触系统的业务同学、要做正式交付的项目成员，以及后续需要培训和复盘的维护人员。它不会只讲功能清单，而是按真实使用流程去讲“先看什么、先做什么、什么时候发布”。

<Aside type="tip" title="推荐阅读顺序">
先读 [系统整体介绍](/docs/system-overview/) 和 [典型使用情景](/docs/scenarios/)，再进入 [5 分钟快速上手](/docs/quickstart/)。如果已经明确自己要做哪类页面，可以直接跳到 [故事地图路线](/docs/story-builder/) 或 [数据驱动路线](/docs/data-builder/)。
</Aside>

## 按你的起点选阅读路径

<Tabs>
  <TabItem label="没有现成数据">
    如果你更关心主题表达、汇报展示和故事脉络，先看 [故事地图路线](/docs/story-builder/)。第一次生成时先要求页面结构和叙事节奏，再逐步补风格、章节和地图联动。
  </TabItem>
  <TabItem label="已有 GeoJSON / JSON / Excel">
    如果你已经有结构化数据，先看 [数据驱动路线](/docs/data-builder/)。重点不是“一次做全”，而是先保证最小可用数据被正确读取，再继续增强筛选、图例和统计模块。
  </TabItem>
  <TabItem label="已有第一版结果">
    如果你已经得到初版页面，直接进入 [修改迭代与质量保障](/docs/quality/) 和 [分享、公开与管理](/docs/publish/)。这一段更适合收口、审阅和正式交付。
  </TabItem>
</Tabs>

## 这套系统的完整闭环

<CardGrid>
  <Card title="1. 输入需求" icon="pencil">
    用自然语言描述主题、结构、地图内容和预期效果。
  </Card>
  <Card title="2. 可选上传数据" icon="seti:json">
    根据需要补充 GeoJSON、JSON、Excel 等结构化数据。
  </Card>
  <Card title="3. 生成并预览" icon="play">
    系统生成页面后，会在工作区里直接运行和预览。
  </Card>
  <Card title="4. 继续修改" icon="refresh">
    你可以继续追问，让系统在当前结果上做增量修改。
  </Card>
  <Card title="5. 自动修复与巡检" icon="check-circle">
    运行错误和明显的视觉异常会进入自动修复与巡检链路。
  </Card>
  <Card title="6. 分享与管理" icon="share">
    最终发布分享链接，并决定公开或仅内部访问。
  </Card>
</CardGrid>
