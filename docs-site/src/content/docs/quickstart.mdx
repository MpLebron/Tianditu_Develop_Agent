## 最短操作路径

<Steps>

1. 进入工作区。

   通常从首页的“开始使用”进入，先确认你已经能看到聊天区、地图预览区和代码区。

2. 描述你想要的页面。

   例如：

   ```text
   请生成一个展示北京中轴线故事的地图页面，
   左侧需要有时间线和地点说明，
   右侧地图要跟随章节切换联动视角。
   ```

3. 等待第一版页面生成并进入预览。

   第一次的目标是确认页面结构方向正确，而不是马上抠细节。

4. 继续补充要求。

   如果第一版还不够完整，继续提出“风格更庄重”“加图例”“补路线说明”“内容更适合汇报”等要求。

5. 页面稳定后再生成分享链接。

   到达可展示状态后，再进入分享步骤补标题、描述和公开方式。

</Steps>

## 可以直接复用的需求模板

```text
请生成一个地图故事页面，要求：
1. 页面包含标题区、左侧内容面板、右侧地图和图例；
2. 地图需要围绕主题地点展开，支持章节切换时联动视角；
3. 页面整体风格克制、清晰，适合分享和展示；
4. 如果需要补充说明，请把关键信息整理成卡片或时间线。
```

<Aside type="caution" title="第一次使用最常见的误区">
不要把第一次生成当成最终交付。更好的做法，是先确认结构、地图范围和主要内容都对，再逐轮补风格、文案和交互。
</Aside>
