# Component Polish Checklist

页面“功能能用但显得粗糙”时，按此清单补齐。

## 1. 状态覆盖

每个核心组件至少覆盖三种状态：

- `default`
- `hover`
- `active` 或 `focus`

数据相关区域必须覆盖四态：

- `loading`
- `ready`
- `empty`
- `error`

## 2. 交互反馈

- 点击后 100ms 内有视觉反馈（高亮/按下态）
- 切换筛选后，列表和地图状态同步
- 错误提示可读、可恢复（有“重试”）

## 3. 对齐与节奏

- 图标、标题、说明文字左对齐基线一致
- 同类卡片高度尽量接近
- 行动按钮在卡片中的位置一致

## 4. 可访问性

- 对比度达标（正文与背景至少 4.5:1）
- 可键盘聚焦（`:focus-visible` 清晰）
- 不依赖颜色作为唯一状态信号

## 5. 地图场景专项

- 浮层不要遮挡核心地理要素
- 地图上的点/线/面颜色与面板图例一致
- Popup 内容紧凑，不塞长段文本
