# UI Planning Workflow

用于把“美化页面”变成可执行方案，避免直接写代码导致反复返工。

## 1. 五段式规划（先写这 5 段，再写代码）

1. **目标用户与任务**
   - 谁在用页面（管理人员/分析人员/公众用户）
   - 核心任务是什么（查询、筛选、对比、应急决策）
2. **信息层级**
   - 一级信息：地图主内容
   - 二级信息：筛选、搜索、图层控制
   - 三级信息：说明、帮助、次要统计
3. **布局骨架**
   - 选择 `左栏 + 右图`、`上方 Hero + 下方案例`、`地图 + 浮层面板` 中一种
4. **视觉系统**
   - 先定义变量，再写组件
5. **状态与反馈**
   - `loading / ready / empty / error` 四态

## 2. 输出模板（给用户看的规划文本）

```md
UI 方案摘要
- 页面目标：
- 目标用户：
- 信息层级：
- 布局骨架：
- 视觉变量：
- 交互状态：
- 移动端策略：
```

## 3. 代码前检查

- 是否先输出了 UI 方案摘要
- 是否定义了 `:root` 视觉变量
- 是否有统一间距体系（例如 8px 基线）
- 是否覆盖四态
- 是否有移动端断点（至少一个）
- 若任务包含地图渲染/数据加载：是否已补齐核心地图技能（如 `map-init`、`bindGeoJSON`、点/线/面图层技能）
- 注意：UI 规划只负责“先规划后编码”，不能替代地图 API 与数据渲染技能
